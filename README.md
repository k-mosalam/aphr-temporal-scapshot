# APHR Temporal Snapshot

This repository hosts an interactive HTML snapshot generated by the **Automated Post-Hazard Reconnaissance (APHR)** system.

The current briefing demonstrates the new **temporal analysis** functionality developed for APHR. Instead of presenting only a single reconnaissance snapshot, the system compares information across multiple automatically generated snapshots to identify evolving observations, repeated signals, changes in reported impacts, and emerging recovery/resilience patterns.

## Live Page

The interactive briefing is available here:

https://k-mosalam.github.io/aphr-temporal-snapshot/

## Current Demonstration

This demonstration focuses on the **2026 Kablalan, Philippines earthquake** and presents an automated preliminary virtual reconnaissance briefing.

The temporal analysis currently covers:

* earthquake impact information collected across multiple snapshots;
* building damage observations;
* infrastructure impact observations;
* resilience and recovery information;
* temporal tracks showing how specific observations evolved over time;
* source and evidence notes supporting the generated briefing.

## About APHR

**APHR** stands for **Automated Post-Hazard Reconnaissance**. The system is intended to support rapid post-hazard information synthesis by collecting, organizing, and summarizing publicly available information after major hazard events.

The goal is not to replace expert reconnaissance, but to provide a preliminary, automatically generated briefing that can help researchers, engineers, emergency-response partners, and domain experts quickly identify important reported impacts and evolving information.

## Temporal Analysis

The temporal analysis feature compares multiple system-generated snapshots over time. In this demonstration, the analysis is based on **12 snapshots**, selected as a balance between information coverage, computational efficiency, and cost.

The system is currently configured to generate snapshots automatically every day at **9:00 p.m. local time**.

## Interactivity

The HTML page is interactive. Users can:

* navigate using the contents menu;
* expand supporting evidence;
* review temporal tracks;
* inspect source and evidence notes;
* explore how observations changed across snapshots.

## Project Status

This is an active research and development prototype. The current version is intended for feedback from potential users and domain experts, especially regarding:

* front-end design;
* usability;
* clarity of temporal evidence;
* usefulness of the temporal analysis;
* possible improvements for future APHR briefings.

## Notes

The briefing is generated automatically, end-to-end. Because the output is preliminary and based on available online information, it should be interpreted as an automated reconnaissance aid rather than a final verified damage assessment.
